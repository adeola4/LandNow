1) WHAT THIS IS
This quiz is a 3-layer binary decision tree used as a lead magnet before the membership application. Its ONLY job is to segment visitors and capture their email. It is NOT a survey.

2) CURRENT QUIZ SHAPE
- Cover: "Which LandNow prospect are you?" + Start button
- Layer 1: 1 question, 2 options
- Layer 2: both branches each have 1 question, 2 options = 4 leaf paths
- Layer 3: all 4 paths converge to 1 result decision = 4 final segments
- Email gate: AFTER Layer 2, BEFORE Layer 3
- Progress indicator: simple step counter

3) 4 SEGMENTS + OUTPUT FIELDS
Each leaf path must map 1:1 to exactly one segment. Output goes to Google Sheet "Entries" via /api/submit-entry as FunnelCategory plus all quiz answers.

A) The Operator — wants active deal flow, already acquires land/water.
B) The Strategist — wants frameworks, in deal-analysis mode, not yet deploying capital.
C) The Buyer — has capital, no operator network, needs deal flow + vetting.
D) The Tire Kicker — wrong audience (passive residential, no real assets, no liquid capital).

5) ADAPTATION RULES — EVERY 10 SUBMISSIONS
Rule A: Find the lowest completion_rate leaf path. Draft ONE new question/option variant for the highest-drop-off step in that path only.

Rule B: Keep the same intent. Do not change the question topic. Change format, wording, or add one more relevant option if currently binary.

Rule C: AB test mode: wrap the variant in a feature flag or URL param "variant=X". 50/50 split via sheet column or client-side random. Do not deploy winner until conversions lift and sample_size >= 30.

Rule D: After adopting a winner, immediately archive the losing variant in a sheet named QuizVersionHistory with columns: version, active, replaced_by, replaced_at, reason. Do not delete old variants — keep the tree history.

Rule E: Cap total variants at 3x current node count. If Layer 1 has 4 variants, max slots = 12. Hard stop when full.

Rule F: If no leaf path has Completion Rate < 35%, do NOT change anything. Changing running winners kills conversion.
